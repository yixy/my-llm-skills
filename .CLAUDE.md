## 行为规则（最高优先级）

每次代码操作后必须执行以下检查。Hooks 会辅助提醒，但最终由你主动遵守。

1. **Wiki 更新** — 使用 Edit/Write 修改 `src/`、`third_party/` 目录下或 `main.go` 文件后
   检查是否有业务逻辑、API 接口、数据库 schema 变更 → 同步更新 `wiki/` 下对应文档（每个页面需有 YAML Frontmatter，含 `title`、`last_updated`，变更时更新其时间戳）：
   - 功能变更 → `wiki/overview.md`
   - 架构调整 → `wiki/architecture.md`
   - DB schema 变更 → `wiki/reference/data-model.md`
   - 每次变更 → `wiki/log.md` 追加摘要记录

2. **单元测试** — 新增/修改功能代码后
   运行对应模块测试确保通过；新增功能代码必须同步添加单元测试用例，覆盖率 ≥ 70%

3. **设计规范** — 涉及以下操作时，必须主动阅读 `docs/rules/design-rules.md` 对应章节：
   - 建表 / DDL 变更 → "建表时适用"
   - API 接口设计 → "API 设计时适用"
   - 新增模块 → "模块设计时适用"
   - 业务逻辑编写 → "程序逻辑编写时适用"
   - JSON/DTO 定义 → "JSON 报文时适用"
   - 批处理任务 → "批处理设计时适用"

4. **编码规范** — 编写 Java/Vue/数据库 代码时，必须遵循 `docs/rules/coding-rules.md` 对应章节的强制规则

## 其他要求

- 优先使用 Java 和 Vue 的 LSP 进行代码搜索和跳转
- 所有 API 接口需定义清晰的请求/响应 DTO，避免直接暴露实体
- 敏感配置（密码、密钥）必须通过环境变量或密钥管理服务注入，不得提交到版本库
- Using --no-gpg-sign consistent with git commit
